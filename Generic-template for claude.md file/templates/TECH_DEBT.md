# TECH_DEBT.md

> Append-only log of known shortcuts, deferred fixes, and accepted risks.
> Never delete entries. If debt is resolved, mark it resolved with a date — do not remove it.
> Updated by Claude when a shortcut is knowingly taken or a fix is explicitly deferred.

---

## Format

```
## [YYYY-MM-DD] — Short title
**What:** What the shortcut or gap is.
**Why deferred:** Why it wasn't fixed now.
**Risk:** What could go wrong if left unaddressed.
**Resolved:** [YYYY-MM-DD] or "Open"
```
